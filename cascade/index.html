Cascade Drupal Theme Documentation

last updated by Margots from DesignsSquare.com Feb 14, 2014

Installation

Whats included?

There are module cascade_theme containing two submodules - designssquare_com_theme_cascade_structure and designssquare_com_theme_cascade_data. The designssquare_com_theme_cascade_structure contains the structure required for Cascade theme to function. The designssquare_com_theme_cascade_data contains sample data and is not required for theme to function

How To Install?

In the following are the steps to install Cascade theme

  1. Copy module cascade_theme module into sites/all/modules folder
  2. Copy themes "boostrap" and "cascade" into sites/all/themes folder
  3. Ensure module 'jquery_update 7.x-2.x-dev' is present, enabled and set to version 1.10(admin/config/development/jquery_update)
  4. Enable theme 'bootstrap'
  5. Enable and set default theme 'cascade'
  6. Enable module designssquare_com_theme_cascade_structure. Ensure its not override state. if so, re-enable till its it in the default state
  7. Enable module designssquare_com_theme_cascade_data if you wish sample data(recommended). Ensure its not in override state. if so, re-enable till its it in the default state
  8. Enable module designssquare_com_menu_importer_cascade to fix main and user menu. Note: this will clear current Main, user menu
  9. clear all cache

About

Overall

All pages that has a particular dependencies(i.e. js,css) is in their own content type(i.e inbox, gallery,etc.) All other pages are of type 'Basic Page'

All content types come with two custom fields - page_title and page_info. Page title field displays the title and it can contain icon tags. The page_info is a little discription or intro of the page that opens up once user clicks on 'info' icon

For carousel to work, you have to copy images accordingly that are currently in 'themes/cascade/images/carousel' folder

All the images and the design is stored in the /design folder. You will have to copy images accordangly

About Menus

To have icons in the the menu item, make sure you add the html icon tags in the menu item title (i.e. Calendar). Also ensure you select 'Allow html' for the menu item containing the icon, thus, html

To have a drop down menu functional, ensure that the menu item with the drop down items is configured with 'show as expanded' setting or checked

For user menu the menu item 'log out' has to be the last one in the menu

About Regions

There are two new regions - 'Pate Top Bar' and 'User Popover'. The 'Page Top Bar' is an area on right of the page title used for search block but can be for any other block.

The 'User Popover' is the region that opens open when clicked on the user name on the right side of the Theme header. A typical usage for this region is to assign User Menu

Dependent Modules

  • menu_html : to allow html in menu items
  • context : to reuse blocks such as search block,etc

Troubleshooting

1. Menu item shows up in html tags
Make sure you have set 'Allow html' for the particular menu element that can be done either at the Page Options or Menu/Edit Menu Item

2. Menu item is not dropping down
Make sure the menu item with sub menu is set to 'Show as expanded' in the Menu configurations

3. Cannot Revert Designssquare_com_theme_cascade_structure module Make sure the Themes - Bootstrap and Cascade is enabled. There are variables in the Designssquare_com_theme_cascade_structure module that require both Themes enabled in order revert
